# 缺陷截图 → Bug 单

## 一、何时触发

以下任一条件满足即触发本技能：
- 用户上传了缺陷相关截图（png / jpg / 聊天附件）
- 用户要求"整理 Bug""识别 Bug 单""写缺陷描述""写缺陷分析"
- 用户说"帮我提 Bug""这个问题帮我写一下"

## 二、输入规范（引导用户提供）

### 2.1 必须输入
- **缺陷截图**（至少一张）

### 2.2 建议输入（结构化补充）

如果用户提供了以下信息，优先采用用户描述的内容：

```

【模块】xxx（如：我的信息 → 我的套餐）

【环境】测试环境 / 正式环境

【账号】xxx

【版本】v1.x.x

【简述】做了什么操作，出了什么问题

```

如果用户未提供，从截图中推断并标注"根据截图推断"。

### 2.3 可选输入（按需 @ 引用，节省 token）

- `@docs/prd/模块名.md` — 相关模块的 PRD 摘要
- `@docs/testcases/TC-模块名.md` — 相关模块的测试用例
- `@Codebase` — 语义搜索项目文档

> **重要**：不要主动加载全量 PRD 或用例文档。仅在用户 @ 引用时读取，
> 或在用户明确要求"结合需求/用例"时，提示用户 @ 对应模块文件。

## 三、识别两类版式

| 版式 | 图中常见结构 | 输出章节 |
| --- | --- | --- |
| **A. 缺陷描述** | Bug 标题、前置条件、复现步骤、实际结果、预期结果 | 模板 A |
| **B. 缺陷分析** | 缺陷分析表格、优化建议列表、严重等级 | 模板 B |

一张图可能只有 A、只有 B，或两者都有：有的则输出，没有的整块省略。

## 四、输出格式要求

1. **语言**：与截图正文一致（一般为简体中文）。
2. **列表**：前置条件 / 实际结果 / 预期结果用无序列表 `- `；复现步骤、优化建议用有序列表 `1. `。
3. **标题层级**：使用模板中的 `##` / `###`。
4. **模块名**：若截图标题含 `【模块】`，原样保留；若无且可推断，补 `【模块】` 并标注"模块名为根据截图推断"。
5. **敏感信息**：手机号、账号等照抄截图；若用户要求脱敏，再改为「已脱敏」。
6. **多图场景**：按上传顺序编号（截图1、截图2…），合并到同一条 Bug 单，复现步骤里标注"见截图X"。
7. **不要编造**：截图中看不清或未出现的信息，用简短说明标出（如「截图模糊：xxx 无法辨认」）。

## 五、输出模板

### 模板 A：Bug 标题 + 缺陷描述 + 禅道字段

```

## Bug 标题

【模块名】一句话概括问题（与截图一致）

## 禅道字段

- **所属产品**：星联应急保障平台 / 星联应急叫应小程序
- **所属模块**：（从截图/描述获取）
- **影响版本**：（从描述获取，未提供则标"待确认"）
- **测试环境**：（从描述获取）
- **严重程度**：S1 / S2 / S3 / S4
- **优先级**：P1 / P2 / P3
- **缺陷类型**：功能缺陷 / UI缺陷 / 数据一致性 / 兼容性 / 性能
- **指派给**：（待指派）

## 缺陷描述

### 前置条件

- …

### 复现步骤

1. …
2. …

### 实际结果
1. …

### 预期结果
1. …

```

**严重程度判断参考**：
- **S1**：系统崩溃、数据丢失、核心链路（SOS 求救）完全不可用
- **S2**：主流程阻塞但有临时替代方案
- **S3**：功能异常但不影响核心业务
- **S4**：UI 体验优化、文案错误等

### 模板 B：缺陷分析 + 优化建议 + 严重等级

```

## 缺陷分析

| 缺陷维度 | 问题描述 | 优化方案 |
| --- | --- | --- |
| … | … | … |

## 优化建议

1. …
2. …

## 严重等级

**等级**：Px（或截图中的写法）

（截图中严重等级下的说明段落，完整复述。）

```

## 六、交付顺序

1. 同时存在 A + B → 先输出 **模板 A**，空一行，再输出 **模板 B**。
2. 图中只有分析页 → 只输出模板 B，开头说明"本截图为缺陷分析版式"。
3. 末尾附 **自检清单** 和 **提取说明**。


## 七、提取说明（每次输出末尾附上）

```

## 提取说明

- **置信度**：高 / 中 / 低
- **模糊字段**：有 / 无（如有，列出具体字段）
- **建议核对**：xxx

```

## 八、项目术语表（自动识别时参考）

| 术语 | 含义 |
| --- | --- |
| 天通救援应急棒 | 卫星应急终端设备 |
| 求救群聊 | SOS 触发建的救援专用群，与"普通通信"并行 |
| 短音 | 语音条数套餐 |
| 报位 | 位置上报套餐 |
| 设备池 | 以设备为维度的套餐池 |
| 倒欠 | 余额可为负但不阻断上报 |
| A 类账号 | 设备管理员 |
| E 类账号 | 紧急联系人 |
| C 类账号 | 普通联系人 |

> 如有新术语，请在此表中补充，避免 AI 误解业务概念。

## 九、不必做的事

- 不主动加载全量 PRD / 用例（等用户 @ 引用）
- 不要求用户跑脚本或安装工具
- 不主动扩展成测试方案或长篇根因分析（除非用户另提要求）
- 不编造截图中不存在的信息
```

```
