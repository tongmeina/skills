# 测试报告编写指南

## 输入素材

- **测试结果.png**：测试模块覆盖数据
- **缺陷汇总.txt**：缺陷详情及优先级
- **潜在风险.png**：风险维度参考

## 输出结构

### 一、测试结果

**核心思路：先分析缺陷汇总，识别问题类型，再动态生成结论**

**步骤1：读取缺陷汇总文件**
- 统计各优先级的缺陷数量
- 按模块分类统计缺陷分布
- 识别问题关键词（如：翻译、i18n、白屏、报错、截断、失效等）

**步骤2：归纳问题类型**
- 根据缺陷内容自动归纳分类，常见方向如：翻译/国际化、UI/排版、功能异常、易用性、数据准确性等
- **具体分类由实际缺陷内容决定，不固定**

**步骤3：生成测试结果**
采用**方案B格式**：
> 本轮测试已完成XX模块的功能验证，发现主要问题集中在 **【动态识别的分类1】 / 【动态识别的分类2】** 等方向，其中 **X 个高优问题需优先修复**，其余问题可迭代优化。
>
> - **【分类1】**：根据缺陷归纳的具体描述
> - **【分类2】**：根据缺陷归纳的具体描述

### 二、修复问题汇总

从缺陷汇总中按优先级排序，**只取前10条**（超过10条则截断，不再列出），底部附动态统计：
> 测试问题汇总：1级：X个；2级：X个；3级：X个；4级：X个，一共：X个，剩余X个未解决（此处统计全量，列表仅展示前10条）

## 注意事项

- **优先读取缺陷汇总**，所有结论必须来源于此，不得自行添加内容
- **避免使用表格**，全部文字描述，方便一键复制
- 量化精准，还原缺陷原貌；功能异常需说明**影响用户的具体场景**
- 领导关注重点：**结论前置 + 问题精炼**，避免繁杂
